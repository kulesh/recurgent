# Code of Conduct

## Our Standard

This project expects respectful, constructive collaboration focused on technical quality and user value.

## Expected Behavior

1. Be specific, honest, and respectful in technical discussion.
2. Accept review feedback and iterate in good faith.
3. Keep contributions aligned with project scope and process.

## Unacceptable Behavior

1. Spam or flooding maintainers with low-value or repetitive pull requests.
2. Automated submission patterns intended to bypass normal review.
3. Repeatedly ignoring maintainer contribution process after warning.
4. Harassment, personal attacks, or discriminatory language.
5. Disruptive behavior that blocks collaboration.

## Enforcement

Maintainers may remove content, close PRs/issues, and restrict participation for repeated or severe violations.

## Reporting

Report conduct concerns through repository maintainers via private channel defined in `SECURITY.md` or repository contacts.
